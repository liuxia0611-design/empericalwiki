# 第 6 章 · 第一篇文献的摄入：5 分钟里发生了什么

**本章你会读到**

- 一篇 PDF 从硬盘进入 EmpiricalWiki 之前，研究者要做的事
- `/empirical-ingest` 这个命令在 5 分钟里实际执行的步骤
- 摄入完成后 wiki 多了哪些文件、每个文件长什么样
- 摄入过程里研究者需要参与判断的几个关键时刻

---

## 6.1 准备工作：把 PDF 放对地方

第一步是把待摄入的论文 PDF 放到 `raw/papers/` 目录。

回忆一下第 3 章讲的：raw 目录是只读的。但"只读"是指**摄入流程不能修改它**——研究者自己往里面放新文件是允许的，这是 raw 这一层的输入入口。

这一步没有特别之处。打开访达，把下载的 PDF 拖进 `raw/papers/`，结束。EmpiricalWiki 不要求 PDF 的文件名按特定规则命名，但建议命名时包含作者和年份，方便后续在终端里识别。

```bash
$ ls raw/papers/
代飞-2025-耐心资本如何影响企业双元创新.pdf
```

可以选地，也可以在 `raw/notes/` 里留一段研究意图的说明，告诉 EmpiricalWiki 你为什么要读这篇。这一步不是必须的，但会影响摄入时模型的判断重点——比如你的研究意图侧重"机制识别"，模型在摄入时会更精细地处理 mechanisms 这一类实体。

```bash
$ cat raw/notes/research-intent.md
我目前在做耐心资本对企业 ESG 表现的研究。
摄入这篇论文是为了：
1. 理解耐心资本的标准测算方法
2. 学习"中介效应"在这个领域怎么做
3. 看作者用了哪些识别策略
```

准备工作就这两件事。研究者花了不到 1 分钟。

## 6.2 触发摄入：一行命令

接下来打开 Claude Code，进入 EmpiricalWiki 项目目录，输入命令：

```bash
$ claude
> /empirical-ingest raw/papers/代飞-2025-耐心资本如何影响企业双元创新.pdf
```

按下回车。研究者的工作暂时告一段落。接下来的 5 分钟，模型在干活。

这 5 分钟里发生了什么？模型按照 `.claude/skills/empirical-ingest/SKILL.md` 里写的步骤，分阶段执行了一系列子任务。下面把每个阶段的内部动作展开。

## 6.3 阶段一：读论文（约 30 秒）

第一阶段是模型把 PDF 读完。

```
[01:00:03] 读取 raw/papers/代飞-2025-耐心资本如何影响企业双元创新.pdf
[01:00:08] PDF 解析完成，正文 14,372 字
[01:00:09] 识别论文结构：摘要、引言、文献综述、研究假设、研究设计、
           实证结果、稳健性检验、结论
```

这一阶段模型只是在"读"，还没有写任何 wiki 文件。它在脑子里把论文的整体结构梳理清楚——哪一段讲研究问题、哪一段讲变量、哪一段讲模型——为后面的拆分做准备。

> [!NOTE]
> **PDF 解析的实际机制**
>
> EmpiricalWiki 用的是 Claude 模型自带的 PDF 阅读能力。模型可以直接读 PDF 文件（包括其中的图、表、公式），不需要先把 PDF 转成纯文本。这意味着论文里的 Table 3 模型回归结果、Figure 2 机制图等等，模型都能看见。

## 6.4 阶段二：识别已有实体（约 20 秒）

第二阶段是模型对照 wiki 当前状态，识别"这篇论文涉及的实体里，哪些已经存在、哪些是新的"。

```
[01:00:35] 扫描 wiki/ 当前状态
[01:00:38] 已存在实体：
           - variables/patient-capital.md
           - datasets/csmar.md
           - models/twoway-fixed-effects.md
           - identification/firm-fixed-effects.md
[01:00:42] 待新建实体：
           - variables/dual-innovation.md
           - variables/managerial-myopia.md
           - mechanisms/myopia-mediation.md
           - hypotheses/H1-pc-promotes-exploration.md
           - hypotheses/H2-pc-inverted-u-exploitation.md
           - papers/dai-fei-2025.md
           - tables/dai-fei-2025-table-3.md
```

这一阶段的关键是**避免重复建实体**。如果耐心资本作为构念在 wiki 里已经有了页面，模型不会新建一个，而是会去更新已有的那个。这是变量页面"跨论文累积"特性的实现机制。

## 6.5 阶段三：写 papers/ 卡片（约 40 秒）

第三阶段开始动笔。模型先写论文卡片本身——也就是 `papers/dai-fei-2025.md`。

```
[01:01:02] 创建 papers/dai-fei-2025.md
[01:01:05] 填写 frontmatter（title, authors, year, journal, sample, ...）
[01:01:18] 抽取研究问题、理论假设、核心结论
[01:01:32] 链接到相关变量、数据集、模型、识别策略页面
[01:01:42] papers/dai-fei-2025.md 完成
```

写完之后这个文件长这样（截取主体部分）：

```markdown
---
title: "耐心资本如何影响企业双元创新——基于管理者短视的机制检验"
authors: ["代飞"]
year: 2025
journal: "管理世界"
sample: "2008-2023 中国 A 股上市公司"
sample_size: 33287
research_question: "耐心资本是否能促进企业的双元创新？管理者短视是否是中介机制？"
date_added: 2026-04-12
---

## 研究问题
本文研究耐心资本对企业双元创新（探索式创新与利用式创新）的影响...

## 理论假设
- H1: 耐心资本促进企业探索式创新
- H2: 耐心资本对企业利用式创新呈倒 U 型影响
- H3: 管理者短视在 H1 中起部分中介作用

## 变量设定
- 自变量：耐心资本（[[patient-capital]]，A2 标准差变体测算）
- 因变量：探索式创新、利用式创新（[[dual-innovation]]）
- 中介变量：管理者短视（[[managerial-myopia]]）

## 模型设定
基于 [[twoway-fixed-effects]] 的回归框架...

## 识别策略
[[firm-fixed-effects]]，控制时间不变的企业层面异质性...

## 主要结论
- H1 支持：耐心资本对探索式创新有显著正向影响（β=0.234, p<0.01）
- H2 支持：耐心资本对利用式创新呈倒 U 型，拐点位于 2.7（[[sasabuchi-u-shape]] 检验通过）
- H3 支持：管理者短视中介比例约 27%
```

这就是论文摄入后的"骨架"页面。所有具体细节都通过 wikilink（`[[页面名]]`）链接到对应实体目录的页面。

## 6.6 阶段四：批量写新实体（约 2 分钟）

第四阶段是这一次摄入里工作量最大的部分——批量创建新实体页面。

```
[01:01:50] 创建 variables/dual-innovation.md
[01:02:08] 创建 variables/managerial-myopia.md
[01:02:25] 更新 variables/patient-capital.md（追加 A2 测算变体）
[01:02:46] 创建 mechanisms/myopia-mediation.md
[01:03:10] 创建 hypotheses/H1-pc-promotes-exploration.md
[01:03:25] 创建 hypotheses/H2-pc-inverted-u-exploitation.md
[01:03:42] 创建 hypotheses/H3-myopia-mediates.md
[01:03:55] 创建 tables/dai-fei-2025-table-3.md
[01:04:08] 更新 datasets/csmar.md（追加引用）
[01:04:15] 更新 models/twoway-fixed-effects.md（追加引用）
[01:04:20] 更新 identification/firm-fixed-effects.md（追加引用）
```

这里每一行的 timestamp 都是模型在写一个文件。看似简单的"创建一个 markdown 文件"实际上要做很多事——

- 确定 slug（变量是 `dual-innovation` 还是 `dualistic-innovation`？参考 wiki 既有命名约定）
- 写 frontmatter
- 写定义段落
- 链接到来源论文（`source_papers: [dai-fei-2025]`）
- 把反向链接也写好（即在 papers/dai-fei-2025.md 里加 `[[dual-innovation]]`）

模型这一阶段做的工作，相当于一个细心的研究助理在边读边记笔记，每读到一个新概念就建一张索引卡，把卡片之间的引用关系建好。

> [!TIP]
> **Literature Variants 章节的更新**
>
> 当 patient-capital 已经存在时，新摄入的论文不会覆盖它，而是会**追加一段新的 Literature Variants**。这一段记录这篇论文用的具体测算方式，附在已有变体之后。第 7 章会专门展开这个累积过程。

## 6.7 阶段五：建立双向链接（约 1 分钟）

第五阶段是最关键也最容易被忽视的一步——建立双向链接。

```
[01:04:25] 扫描所有新建/更新页面，识别 wikilink
[01:04:32] papers/dai-fei-2025.md → 12 个 outbound link
[01:04:38] 检查反向链接对称性
[01:04:45] 在 12 个被链接页面里添加 cited_by 反向链接
[01:04:55] 双向链接对称性验证通过
```

这一步在第 2 章和第 3 章都强调过——任何正向链接都需要对应的反向链接。模型不光要建立这种双向关系，还要**验证它确实建好了**。如果某个反向链接没建上，会在这一步被发现。

## 6.8 阶段六：更新 graph 和 index（约 20 秒）

第六阶段是把这次摄入产生的所有新链接关系汇总到 `wiki/graph/edges.jsonl`，并更新 `wiki/index.md`（wiki 内容目录）和 `wiki/log.md`（编年志）。

```
[01:04:58] 写入 graph/edges.jsonl（新增 36 条边）
[01:05:05] 更新 index.md
[01:05:09] 在 log.md 追加：
           [2026-04-12] empirical-ingest dai-fei-2025
[01:05:11] 摄入完成
```

到这一步，整个摄入流程结束。研究者从输入命令到看到"摄入完成"消息，时间过去了约 5 分钟。

## 6.9 摄入完成后研究者要做什么

摄入完成不等于结束。这套设计有一个核心信条——**研究者必须人工核对摄入产物**。

模型再好也会犯错。可能某个机制路径理解错了，可能某个变量测算没抓全，可能某个识别策略归类不准。这些错误如果不被发现，会变成 wiki 里的"幻觉传播"——错误判断被链接到其他正确判断旁边，整个 wiki 的可信度就被污染。

所以摄入完成后，研究者要做的事是：

1. **打开 papers/dai-fei-2025.md**，从头到尾读一遍
2. 对照原文 PDF，看每一段是否准确
3. 重点检查：变量定义是否抓对、模型设定是否对、机制路径是否抓全、识别策略是否归类准
4. 发现错误，**手工修改 wiki 文件**，或者用 `/edit` 命令让模型按你的指令修改

这个核对过程通常需要 5-10 分钟。**整个"摄入 + 核对"加起来 10-15 分钟**。比研究者自己从头读 PDF 整理笔记要快，但又不是完全自动化——研究者仍然需要参与判断。

> [!NOTE]
> **AI 辅助 ≠ AI 代写**
>
> EmpiricalWiki 这一套流程的边界是清楚的：模型负责把 PDF 拆成结构化卡片，研究者负责确认这些卡片是否准确。**判断权一直在研究者手上**。这是科研伦理的底线——AI 可以加速劳动，但不能替代判断。任何让研究者跳过核对、把摄入产物直接拿去用的做法，都不是 EmpiricalWiki 推荐的用法。

## 6.10 第二篇、第十篇、第二十篇

第一篇论文的摄入花了 5 分钟。第二篇大概也是 5 分钟。第十篇可能稍快一些，因为多数变量、数据集、模型已经在 wiki 里存在了，模型不需要新建，只需要在已有页面里追加一段。

到第二十篇时，单篇摄入时间可能已经压缩到 3 分钟左右。**但更重要的是，wiki 累积出来的价值开始指数增长**。第一篇摄入时，patient-capital 页面只有 1 种测算变体；第二十篇摄入完，可能已经有 8 种变体——这本身就是一份高质量的方法学综述，是研究者自己手动整理可能要花一周时间的工作。

下一章会专门展开这个"累积"过程：variable 页面是怎么从 1 段长到 8 段的，每一段加进来时模型做了什么判断。

---

## 本章小结

`/empirical-ingest` 一篇论文的过程大约花 5 分钟，分六个阶段：读论文（30 秒）、识别已有实体（20 秒）、写论文卡片（40 秒）、批量写新实体（2 分钟）、建立双向链接（1 分钟）、更新 graph 和 index（20 秒）。摄入产物包括论文卡片本身、若干新建的变量/数据集/模型/机制/假设/识别/稳健性/异质性/表格页面、以及所有相关页面之间的双向链接。

摄入完成不等于结束。研究者需要人工核对摄入产物，确保模型的拆分判断是准确的。这是 EmpiricalWiki 设计哲学里不能省略的一步——AI 加速劳动，但不替代判断。
